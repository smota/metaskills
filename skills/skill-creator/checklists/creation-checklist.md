# Skill creation checklist

- [ ] Skill has a clear, narrow purpose.
- [ ] `SKILL.md` has valid frontmatter.
- [ ] Description includes activation triggers.
- [ ] Core workflow is explicit.
- [ ] Detailed material is moved to supporting files.
- [ ] Examples are realistic.
- [ ] Boundaries are documented.
- [ ] Validation or eval cases are included.
- [ ] Required templates live inside the skill package.
- [ ] No secrets or private project data are included.
