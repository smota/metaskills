# Agent builder intake checklist

- [ ] Goal and target users are clear.
- [ ] Current maturity level is stated or marked unknown.
- [ ] Target maturity level is stated.
- [ ] Target harnesses are named or marked unknown.
- [ ] Required, optional, unavailable, and unknown runtime capabilities are captured.
- [ ] Tools/actions are listed, or explicitly `none`.
- [ ] Agent-to-agent collaboration need is yes/no/unknown.
- [ ] Subprocess/job need is yes/no/unknown.
- [ ] Acceptable fallback behavior is captured.
- [ ] Mutation authority and confirmation requirements are captured.
- [ ] Knowledge sources and freshness expectations are captured.
- [ ] Output format and acceptance criteria are captured.
