# Learning workflow

The learning workflow keeps skills improving without becoming a hidden platform.

```text
Create → Eval → Use → Observe → Improve → Benchmark → Release → Learn
```

## Principles

- Learning is opt-in.
- Records are file-based and reviewable.
- Lessons should become concrete changes or eval cases.
- Avoid collecting sensitive data.
- Prefer summaries over raw transcripts.

## Process

1. Capture an observation in `skill-learning-log.md`.
2. Link evidence such as failed eval, user feedback, or benchmark report.
3. Decide whether the observation requires an improvement.
4. Add or update eval cases before changing behavior when practical.
5. Benchmark meaningful changes against the previous version.
6. Record the released change and remaining hypotheses.
