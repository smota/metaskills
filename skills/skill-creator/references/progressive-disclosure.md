# Progressive disclosure

Progressive disclosure keeps skills efficient and easier to maintain.

## Rules

- Metadata and description should be enough for activation.
- `SKILL.md` should contain only core workflow and essential rules.
- Detailed procedures belong in `references/`.
- Reusable forms belong in `templates/`.
- Review gates belong in `checklists/`.
- Examples belong in `examples/`.

## Why it matters

- lowers context cost before activation
- makes the main skill easier to review
- lets agents load details only when needed
- supports continued improvement without bloating `SKILL.md`
